# Lodestar Documentation

This documentation is built using [Docusaurus](https://docusaurus.io/) and is published at [chainsafe.github.io/lodestar](https://chainsafe.github.io/lodestar/).

## Versioning

Documentation supports versioning so users can view docs for specific releases. A version dropdown in the navbar allows switching between versions. The latest stable release is always the default.

### How it works

Versioned docs are **not stored in source control** — they're maintained on a separate `docs-versions` branch and fetched at build time by CI.

**Automated workflows:**

- **`docs-version.yml`**: Triggers on release tags (`v*`). Creates a Docusaurus version snapshot and pushes to the `docs-versions` branch.
  - **Stable releases** (e.g., `v1.40.0`) automatically replace matching pre-release versions (`1.40.0-rc.*`)
  - Keeps the last 10 stable versions; older ones are pruned automatically
- **`docs.yml`**: Deploys the docs site. Triggers on:
  - `unstable` / `stable` pushes
  - After "Version docs" workflow completes — picks up new releases
  - Manual `workflow_dispatch`

**Pre-release lifecycle:**

```
v1.41.0-rc.0 tag → dropdown: 1.41.0-rc.0 | 1.40.0 (default) | ... | Next 🚧
v1.41.0 tag      → dropdown: 1.41.0 (default) | 1.40.0 | ... | Next 🚧
                              (1.41.0-rc.0 automatically removed)
```

Local development works without any versioning files present — `docusaurus.config.ts` reads `versions.json` dynamically and falls back gracefully.

### Manual versioning (if needed)

`npx docusaurus docs:version` snapshots the current docs sources.

For Lodestar, some docs (CLI references, package READMEs, etc.) are generated by a **repo-root** script (`pnpm docs:build`). Run it first to include generated docs in the snapshot.

```bash
# Refresh generated docs (run from repo root)
pnpm docs:build

# Create version snapshot (run from docs/)
cd docs
npx docusaurus docs:version <VERSION>
```

## Development & Build

From the repository root, set up the docs environment:

```bash
cd docs
pnpm install
```

Start the local development server. Most changes are reflected live without having to restart the server.

```bash
pnpm start
```

Build the static site into the `build` directory:

```bash
pnpm build
```
